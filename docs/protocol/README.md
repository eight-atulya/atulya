# ATULYA Protocol

This folder is the canonical home for the ATULYA protocol design.

## Reading order

1. [`design-direction.md`](design-direction.md)
2. [`record-model.md`](record-model.md)

## What belongs here

Use this folder for material that defines the durable protocol shape of ATULYA, especially:

- canonical record invariants
- record families
- typed fields and schemas
- governance and policy semantics
- audit and integrity semantics
- extension and integration rules

## What does not belong in the top-level README

The following should stay out of `README.md` unless they are summarized briefly and linked here:

- full protocol specifications
- long schema examples
- detailed record definitions
- taxonomy tables
- governance field inventories

That separation keeps the repository legible while letting the protocol grow in a structured way.
