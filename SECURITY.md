# Atulya Security Policy

## Supported Versions

Atulya actively supports and releases security patches for the latest version only. Our triage and patch process prioritizes critical vulnerabilities as classified by the CVSS v3.x standard.

| Version | Actively Supported   |
| ------- | ------------------- |
| latest  | ✅                  |

Older releases and forks outside the mainline branch are not monitored for security updates.

---

## Vulnerability Reporting

If you discover a security vulnerability, **please report it privately** to the Atulya maintainers so we can take prompt action and minimize potential risk to users. The recommended method is via a [GitHub Security Advisory](https://github.com/eight-atulya/atulya/security/advisories/new).

- **Response Time:** We aim to acknowledge your report within 48 hours.
- **Remediation:** When validated, we will assess and address the issue as a top priority. Fix timelines are determined by impact and complexity—critical issues may see hotfixes within days.

### What to Include

Your vulnerability report should contain, where applicable:

- Vulnerability type (e.g., RCE, privilege escalation, XSS, SQL injection, logic bug, etc.)
- Relevant source file names and line numbers
- Commit SHA, tag, or URL identifying the affected code version
- Reproduction steps (minimal and deterministic if possible)
- Any special configuration or environment required to reproduce
- Proof-of-concept code and/or exploit (if available)
- Impact assessment—what is at risk, possible attacker actions, and potential exposure

These details help us efficiently confirm, reproduce, and resolve the reported issue.

---

## Communication Language

Please submit reports in **English** for fastest processing. You may also encrypt sensitive submission using our GPG key (contact maintainers).

---

## Vulnerability Disclosure Policy

We adhere to [Coordinated Vulnerability Disclosure](https://www.cisa.gov/resources-tools/programs/coordinated-vulnerability-disclosure-program) guidelines. We work in good faith with security researchers; responsible and discreet reporting will never be subject to legal retaliation, and researcher credit is given in all published advisories and release notes upon request.

Thank you for contributing to the security and trustworthiness of Atulya.
